`compound::property` serves as a "category of primitives" where objects are variables of primitive types.
This namespace maps to and from properties of pure compounds, mostly using corresponding states method. 
However, it excludes mappings that use group contribution methods. 

Due to memory constraints, we can only ever represent a very limited set of the most commonly occuring natural compounds. 
These compounds tend to be simple in structure, so group contribution methods aren't typically geared toward them. 
Group contribution methods tend to require too much parameterization for the amount of data we can get back out from them 
for use within the model. However the Joback method might be simple enough to consider adding as a separate category/namespace.
